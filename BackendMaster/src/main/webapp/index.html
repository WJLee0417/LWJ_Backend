🚀 Backend Master 통합 프로젝트 가이드

Servlet, JSP, Session, Cookie, Filter, JSTL이 모두 적용된 완성형 웹 애플리케이션입니다.

✅ 1단계: 로그인 및 쿠키 테스트

가장 먼저 로그인 페이지로 이동하여 통신이 정상적으로 이루어지는지 확인합니다.

  • 테스트 계정: admin(1234), user01(1111), user02(2222)
  • 실패 방어: 일부러 틀린 비밀번호를 입력하여 Forward를 통한 에러 메시지가 잘 뜨는지 확인합니다.
  • 쿠키(Cookie) 확인: '아이디 저장'을 체크하고 로그인 후, 나중에 다시 접속했을 때 아이디가 남아있는지 체크합니다.

✅ 2단계: 세션 및 데이터 출력 테스트

로그인에 성공하면 게시판 화면(board.jsp)으로 리다이렉트 됩니다.

  • Session & EL: 환영 메시지에 본인의 이름이 정확히 출력되는지 확인합니다.
  • JSTL & Filter: 데이터베이스(MockDB)의 게시글 4개가 표 형태로 출력되며, 한글이 깨지지 않는지 확인합니다.
  • 비정상 접근 차단: 시크릿 모드 등 다른 창에서 board.jsp로 강제 접속을 시도하여 튕겨내는지 확인합니다.

✅ 3단계: 로그아웃 테스트

모든 확인이 끝났다면 화면 우측 상단의 [로그아웃] 버튼을 클릭합니다.

  • 세션(Session) 파기: 로그아웃 후 다시 뒤로가기를 누르거나 board.jsp에 접속하여 세션이 완전히 비워졌는지 검증합니다.

✅ 4단계: CRUD (입력/조회/삭제) 테스트

데이터의 생성부터 소멸까지의 과정을 테스트합니다.

  • Create: [글쓰기] 버튼을 눌러 새 게시글을 추가하고 목록에 나오는지 확인합니다.
  • Read: 서블릿이 DB에서 데이터를 읽어 JSTL로 화면을 구성하는지 확인합니다.
  • Delete: [삭제] 버튼 클릭 시 해당 데이터가 지워지고 목록이 갱신되는지 확인합니다.
  • Redirect 패턴: 글쓰기/삭제 후 주소창이 자동으로 목록 서블릿으로 돌아가는지 체크합니다.
👉 테스트 시작하기 (로그인 화면으로 이동)
