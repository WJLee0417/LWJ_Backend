SECURITY

Mission 01. 내 비밀번호는 안전할까?

  • 회원가입 페이지에서 자신만의 계정을 만들어보세요.
  • 로그인 후 MySQL Workbench를 열어 member_tbl을 조회하세요.
💡 Behind: PasswordUtil이 비밀번호를 64자리 SHA-256 해시값으로 변환했습니다. 개발자도 원문을 알 수 없는 단방향 암호화가 적용되었습니다.
DATABASE

Mission 02. 연쇄 삭제의 마법 (Cascade)

  • 게시글에 댓글을 3개 달고, 그 게시글을 과감하게 삭제해보세요.
  • DB의 comment_tbl에서 해당 댓글들이 어떻게 되었는지 확인하세요.
💡 Behind: ON DELETE CASCADE 제약조건 덕분에 부모 데이터 삭제 시 자식 데이터도 자동 삭제되어 데이터 무결성이 유지됩니다.
LOGIC

Mission 03. 공지는 왜 항상 위에 있을까?

  • 관리자 계정으로 공지사항을 쓰고, 일반 글을 여러 개 써서 2페이지로 가보세요.
  • 2페이지에서도 공지사항이 상단에 고정되어 있는지 확인하세요.
💡 Behind: BoardDAO의 투트랙(Two-Track) 쿼리 로직이 페이징과 무관하게 공지사항을 최상단에 항상 렌더링합니다.
UI/UX

Mission 04. 이빨 빠진 번호 예쁘게 채우기

  • 목록 중간의 글을 삭제한 뒤, 번호가 1, 2, 3, 4 순으로 예쁘게 정렬되는지 보세요.
  • DB의 실제 ID(PK) 값은 어떻게 변했는지 비교해보세요.
💡 Behind: DB의 PK는 고유 식별을 위해 유지하고, 화면에는 가상 번호 공식을 적용하여 사용자 친화적인 순번을 제공합니다.
